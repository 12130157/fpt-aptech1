Sơ đồ sau minh họa mối quan hệ giữa các lớp ta sẽ nói chi tiết trong tute 4


1. Animation (IAnimation, Animation, OrderAnimation, DisorderAnimation)
Mô tả một tập hợp các khung hình theo một trật tự nhất định. Mỗi khung hình có kích thước bằng nhau.
Và được xác định vị trí theo tạo độ (x,y) như hình:

Interface IAnimation định nghĩa các method sau:
/**
    * @return current frame of animation
    */
    public Point currentFrame();

    /**
    * @return true if current frame is first frame
    */
    public boolean isFirstFrame();

    /**
    * @return true if current frame is last frame
    */
    public boolean isLastFrame();

    /**
    * change the current frame to next frame of the animation
    * @return current frame after changed
    */
    public Point nextFrame();
Trong source code ví dụ, có 2 class implement interface này, đó là OrderAnimation và DisorderAnimation.
OrderAnimation mô tả 1 animation theo thứ tự từ khung hình x đến khung hình y (x<y)
DisorderAnimation mô tả 1 animation theo thứ tự được định nghĩa bằng 1 mảng vị trí các khung hình.
Ngoài ra, interface IAnimation cũng định nghĩa 2 kiểu animation, đó là FORWARD_ANIMATION và PINGPONG_ANIMATION
FORWARD_ANIMATION: các khung hình chạy từ 0 -> n và cứ thế lặp đi lặp lại
PINGPONG_ANIMATION: các khung hình chạy từ 0 -> n, n -> 0 và cứ thế lặp đi lặp lại

2. Animator (IAnimator, Animator)
Animator được đưa ra nhằm để quản lí một animation.
Interface IAnimator định nghĩa các method sau:
/**
    * @return name of animator
    */
    public String getName();

    /**
    * play the animation
    * @param g
    * @param x
    * @param y
    */
    public void animate(Graphics g, int x, int y);
Method animate được cài đặt như sau:
public void animate(Graphics g, int x, int y) {
        long currentMillis = System.currentTimeMillis();
        if (currentMillis-m_currentMillis>=m_interval) {
            m_animation.nextFrame();
            m_currentMillis = currentMillis;
        }

        m_animation.paint(g, x, y);
    }
Cứ sau 1 khoảng thời gian m_interval định trước, lần lượt từng frame của animation sẽ được vẽ ra màn hình.

3. Sprite (ISprite, Sprite)
Sprite sẽ bao gồm nhiều animator. Mỗi animator sẽ chịu trách nhiệm biểu diễn 1 trạng thái nào đó của sprite.
Ví dụ như sprite Đấu sĩ, bao gồm các animator như Đi, Đứng, Nhảy, Đấm, Đá...
Interface ISprite định nghĩa các method sau:
/**
    * add new animator
    * @param animator
    */
    public void addAnimator(IAnimator animator);

    /**
    * remove animator
    * @param name
    */
    public void removeAnimator(String name);

    /**
    * @return current animator
    */
    public IAnimator currentAnimator();

    /**
    * change to use new animator
    * @param name
    */
    public void changeAnimator(String name);

    /**
    * play the sprite
    * @param g
    * @param x
    * @param y
    */
    public void sprite(Graphics g, int x, int y);

amd700:

Em chưa hiểu ý anh khi nói đến cái animation clock. Anh có thể nói cụ thể hơn ko?

Đại khái là tự dùng thread để implement 1 cái clock riêng. Thread này sẽ sleep một khoản khắc (interval) định sẵn trước cập nhật time.
Dùng Clock.getTime() giúp mình mình tránh phải gọi System.cuttentTimeMills() nhưng phải chỉnh resolution (sleep interval) sao cho phù hợp với animation của game để khỏi bị giật.
Hoặc dùng ClockListener để được báo "clock tick" mỗi khi clock thức giấc.

Cách nữa là dùng Clock interface để cài strategy riêng - Clock implementation có thể là thread base như trên hoặc System.currentTimeMillis() - tuỳ theo performance.
