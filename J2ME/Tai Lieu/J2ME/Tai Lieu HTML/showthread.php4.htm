2.2 Biến môi trường CLASSPATH

Biến môi trường CLASSPATH là khái niệm đường dẫn rất quan trọng của Java nó ảnh hưởng đến quá trình biên dịch,thực thi ,cấu hình ... và cũng là điều mà rất nhiều lập trình viên Java hay quên nhât. Bạn hãy dành chút thời gian để nắm vững về biến môi trường CLASSPATH này.

Tương tự WIndows thường tìm các file thực thi .exe ở một số nơi như : trong thư mục hệ thống của Windows ,trong thư mục hiện hành của ứng dụng, trong các thư mục trỏ đến bởi biến môi trường PATH

Biến môi trường CLASSPATH với Java cũng tương tự như biến PATH mà Windows sử dụng . Khi bạn yêu cầu Java thực thi một lớp , Java sẽ tìm lớp đó ở thư mục hiện hành , tiếp theo là các thư mục được đặt trong đường dẫn do CLASSPATH chỉ định.

Ví dụ,bạn đang ở thư mục C:\Learing>javaHellOWorld

Do HellOWorld.class nằm ngay trong thư mục hiện hành nên Java tìm ra và cho thực thi . Bây giờ bạn hãy chuyển HellOWorld.class sang thư mục khác (như c:\temp) và gõ lại lệnh:

Mã:
C:\Learning>java HellOWorld
Lần này thư mục C:\Learning không còn file.class nữa và java sẽ báo lỗi sau:

Mã:
Exception in thread "main"
      java.lang.NoClassDefFoundError : HellOWorld
Muốn Java thực thi file.class trong thư mục C:\Temp bạn có thể chỉ ra tên đường dẫn chứa file.class cho biến CLASSPATH như sau:

Mã:
C:\Learning>SET CLASSPATH = CLASSPATH ; C:\Temp
Lệnh trên đặt thêm đường dẫn c:\temp vào biến CLASSPATH hiện hành để Java truy tìm các file.class. Và bây giờ bạn có thể gọi thực thi HellOWorld từ bất kỳ vị trí nào không nhất thiết phải đang đứng ở thư mục chứa file.class

Một đặc tính thú vị khác là Java có thể tìm thấy các file.class trong file.zip. Ví dụ bạn có thể dùng Winzip nén file HellOWorld.zip thành file App.zip sau đó thay vì trỏ vào thư mục c:\temp bạn có thể trỏ đường dẫn vào file.zip như sau:
Mã:
C:\Learning>SET CLASSPATH= CLASSPATH ; C:\App.zip
Cơ chế này rất thuận tiện nếu bạn có rất nhiều file.class hỗ trợ cho một ứng dụng. Tất cả các file.class sẽ được nén vào file.zip và khi đó file.zip của bạn được Java xem tương tự như file.exe của Windows.Đây là một ý tưởng về ứng dụng có khả năng phân tán mã khả chuyển cao.Rất có ích cho việc lập trình game sau này.

2.3 Cấu trúc chương trình Java

Hãy xem qua cấu truc của chương trình Java HellOWorld đơn giản trên đây để bạn có một khái niệm coơbản về xây dựn và thiết kế một ứng dụng kiểu Java.Thích hợp cho việc lập trình game sau này.

Như đã nêu,Java là hướng đối tượng nên tất cả các hàm phương thức đều phải nằm trong khai báo lớp của đối tượng.Kể cả chương trình chính.

Mã:
public class HellOWorld
{
   ...
   }
Trên đây là lớp ứng dụng HellOWorld với từ khoá class cho biết khai báo lớp và public cho biết lớp này là toàn cục cho phép tất cả các lớp khác sử dụng nó. Tuy nhiên lớp không phải là chương trình chính , chương trình chính được đặt bên trong lớp và được cài đặt như một phương thức tĩnh của lớp như sau:

Mã:
  public static void main (String [] args)
  {
          }
Với C/C++ thường chương trình chính là hàm tên main(), với Java phương thức mang tên main() sẽ được xem như là điểm vào đầu tiên của chương trình chính. Khi bạn gọi java HellOWolrld,trình diễn dịch Java sẽ nạp lớp HellOWorld vào bộ nhớ và tìm phương thức mang tên main() thực thi. Đi sâu vào khai báo của phương thức main() bạn sẽ thấy các từ khoá sau:
  • public cho biết main() là phương thức chung có thể được triệu gọi từ môi trường bên ngoài.
  • static cho biết mạin() là phương thức tĩnh (static). Phương thức tĩnh là phương thức có thể triệu gọi thông qua tên lớp mà không cần khởi tạo thể hiện của đối tượng.
  • void cho biết main() là một phương thức không trả về trị
  • String []args là mảng chuỗi chứa đối số truyền trên dòng lệnh khi bạn gọi chương trình Java.Ví dụ khi bạn gọi chương trình Java:
Mã:
        java HellOWorld MinhKhai
Trong phương thức main() lúc này args là mảng chứa 2 giá trị arg[]="Minh" và args[1]="Khai"

Nội dung phương thức main() là tất cả những gì bạn muốn viết nên chương trình.Trong chương trình đơn giản HellOWorld chúng ta chỉ in ra chuỗi "HellOWorld" bằng lệnh:
System.out.println ("HellOWorld");

Đây là cách truy xuất và gọi lệnh kiểu đối tượng trong Java trong đó:
  • System là thư viện hệ thống chứa rất nhiều đối tượng giúp bạn xử lý xuất nhập tính toán
  • out là một đối tượng trong thư viện System
  • println là phương thức của đối tượng out dùng in mỗi chuỗi ra màn hình Console
Bạn hãy thử viết một chương trình thực tế hơn xem sao?chương trình tính 2 số nhập vào từ đối số dòng lệnh

Mã:
   public class AddNumber
 {
         public static void main (String[] args)
  {
                 Integer x=new Integer (args[0]);
                 Integer y=new Integer (args[1]);

           int result=x.intValue () + y.intValue();
 
                     System.out.println("Result="+result);

          }
  }
Bạn lưu nội dung chương trình trên vào file AddNumber.java biên dịch chương trình như sau:
Mã:
 >javac AddNumber.java
Chạy chương trình:
Mã:
>java AddNumber 1 2
Result=3
Chương trình trên đây sử dụng 2 đối số dòng lệnh cộng và in ra kết quả.Nào chúng ta hãy bắt đầu tìm hiểu ngôn ngữ lập trình java.

Chương 3. Ngôn ngữ Java

3.1 Ghi chú (Comment)
Chúng ta rất cần các ghi chú để gợi nhớ lại quá trình suy nghĩ của mình, kiểm nghiệm các lập luận và nhất là khi chúng ta cần bổ sung, sửa chữa , nâng cấp chương trình. Các ghi chú không làm ảnh hưởng đến chương trình, chúng dành cho người lập trình đọc hiểu mã nguồn chi tiết hơn.Khi biên dịch Java sẽ bỏ qua các đoạn ghi chú.
Java hỗ trợ 3 cách ghi chú:
Mã:
Bắt đầu               Kết thúc               Mục đích
  /*                          */                    Đoạn văn bị giới hạn là phần ghi chú
  //                         (none)                Phần câu văn còn lại là ghi chú
  /**                         */                   Phần câu văn bị giới hạn là phần ghi
                                                      chú cho JavaDoc
Cách thứ nhất và cách thứ 2 có nguồn gốc từ C/C++

Cách thứ nhất dùng khi cần ghi chú trên nhiều dòng.
Trích dẫn:
/* Đoạn văn này là phần ghi chú được
chia ra làm nhiều dòng */
Cách thứ 2 ghi chú cho từng dòng lệnh, từ dấu // trở về cuối dòng là đoạn ghi chú:
Mã:
public string s ; //phần ghi chú cho dòng lệnh này
Cách thứ ba là cách ghi chú mới chỉ dùng trong java.Nó giúp cho việc sưu liệu hoá chương trình (program documentation) của trình biên soạn tài liệu JavaDoc.Chúng ta sẽ bàn tới trong chương Các cấu thành của ngôn ngữ Java.

Chú ý:Ta không thể dùng tính xếp lồng để trình bày các ghi chú.Nếu trong cặp /* và */ có dùng ký hiệu // thì ý nghĩa của // bị bỏ qua,điều ngược lại cũng đúng.

Ví dụ chương trình AddNumber trên đây sẽ trở nên dễ hiểu hgơn nếu bạn đặt vào các ghi chú:


Mã:
 
  /*Chương trình Java cộng 2 số
  Các số được nhập vào từ đối số dòng lệnh
  */
  public class AddNumber
   {
      /*Chương trình chính*/
    public static void main (String[]args)
   {
               /* Chuyển đổi số từ chuỗi thành số nguyên */
       Integer x=new Integer (args[0]) ; //số thứ nhất
       Integer y=new Integer (args[1]) ; //Số thứ hai
   
   
        //Lấy về giá trị và cộng ô 1
      int result=x.intvalue () + y.intValue ();
   //In kết quả sau khi tính
       System.out,println("result="+result);
             }
        }
