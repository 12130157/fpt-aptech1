2.4 Thông dịch (Interpreter)

Java là một ngôn ngữ thông dịch (xét trên phương diện thực thì thì Java là một ngôn ngữ thông dịch trên cơ sở đã biên dịch trước thành bytecode) nên nó trở nên khả chuyển.Các ứng dụng Java có thể dễ dàng chạy trên các máy tính với các nền phần cứng khác nhau như Macintosh,Intel,Sun Microsystems,Alpha...chỉ cần đi kèm với các ứng dụng đó là bộ thông dịch,bộ gỡ rối và nhất là bộ thư viện kiểm soát mã thực thi(runtime library).Khi ấy các lập trình viên sẽ được tự do không còn lệ thuộc vào nội dung giới hạn của ngôn ngữ máy,mà chỉ chú ý đến các thư viện module dùng chung phục vụ cho ứng dụng.

2.5 Mạnh mẽ (Robust)

Dụng ý của Java là giúp lập trình viên có thể tạo nên những chương trình chắc chắn,không phạm nhiều vào những lỗi khi chạy (runtime error).Java không cho phép lập trình viên khai báo các biến một cách tuỳ tiện.Biến trong Java luôn phải tường minh.Ngay cả biến kiểu dãy (array) cũng không chỉ là một con trỏ như trong C mà là ngôn ngữ thực.Nhờ đó mà những lỗi thuờng gặp trong khi viết bằng ngôn ngữ C như cấp phát bộ nhớ,bộ nhớ tràn, trùng lặp bộ nhớ,... đã được giải quyết.

2.6 Bảo mật (Security)

Java được viết cho các ứng dụng chạy trên môi trường mạng/phân tán.Do đó Java phải được và đã được thiết kế sao cho có khả năng tạo được một hệ thống no-virus,không bị phá rối. Kỹ thuật xác minh của Java dựa vào cách mã hoá theo khoá công cộng.

Java cũng không hỗ trợ biến con trỏ.Một trình thông dịch bytecode sẽ kiểm tra chặt chẽ các mã byte,không cho bất kỳ hành vi không được phép nào thực hiện.

2.7 Kiến trúc trung tính (Neutral architecture)
Hãy nhìn trở lại những năm tháng đen tối vào cuối thập kỷ 80.Lúc đó người lập trình đã rối mù với biết bao hệ điều hành vô cùng khác biệt được cài đặt trên những máy IBM,Apple,Commodore,Radio Shack, Atari... Các ứng dụng chỉ có thể dùng được trên những hệ máy cụ thể.Đến nay tình hình có cải thiện hơn,số lượng hệ điều hành đã giảm nhưng sẽ không thể nào đồng quy,vì đó là những hệ điều hành của những ông "trùm" Windows,Sun Solaris,UNIX, Macintosh.Và quả là vẫn còn nhiều rối rắm khi nghĩ đến các phần mềm viết trên nền Intel được sử dụng trên nền các bộ xử lý no-Intel (Non-Intel CPU)

Với Java,vấn đề đã trở nên đơn giản,khả năng thông dịch với nguồn mã lệnh đã được biên dịch dưới dạng bytecode cho phép tạo ra các máy Java ảo (Java Virtual Machine-JVM) trên mỗi hệ thống.Các ứng dụng Java sẽ chạy trên nền trung tính của các máy Java ảo đó dễ dàng.Trình thông dịch và hệ Java ảo rất nhỏ để có thể cài đặt trên các máy con mà không tốn nhiều tài nguyên.

2.8 Khả chuyền

Như đã nói ở trên,do tạo được các máy Java ảo tương tích với môi trường được cài đặt tạo nên kiến trúc trung tính,các ứng dụng Java chỉ cần viết sao cho chạy được trên máy Java ảo đó là có thể cài đặt và chạy tốt trên mọi hệ thống ("Write Once,Run Anywhere" khẩu hiệu nổi tiếng của Java bạn có thể hiểu là "Viết 1 lần,chạy khắp mọi nơi").Mặt khác,các kiểu dữ liệu của Java được định nghĩa không phụ thuộc vào bộ xử lý hay vào hệ điều hành mà các ứng dụng được cài đặt.

2.9 Hiệu quả cao (High-performance)

 Rõ ràng so với các chương trình được biên dịch hoàn toàn bằng C hay C++,các chương trình Java không có hiệu quả cao hơn.Nhưng với tính năng giao diện đồ hoạ,sự đơn giản,nhỏ gọn,tính khả chuyển,ta cũng có thể nói là hiệu quả chung của các chương trình Java là đáng khích lệ so với tính phức tạp của C/C++.

Ở đây mình muốn nói đến điểm yếu của các chương trình Java là ở tốc độ thực thi.Nó khá chậm so với các ngôn ngữ khác.Đó là do các chương trình Java phải và chỉ chạy trên các máy Java ảo với chương trình trên một môi trường duy nhất,chúng ta có thể sử dụng việc liên kết với các chương trình nguồn (gốc-native program) như đã từng làm khi kết nối các đoạn ngôn ngữ máy (assembly language) vào các chương trình C/C++.VỚi Java 1.4 bạn không còn phải lo về tốc độ nữa,khả năng thực thi mã byte code của Java được tối ưu hoá với tốc độ thực thi không thua các chương trình C++.Bên cạnh đó,sự nâng cao tính năng của các máy tính cá nhân ngày nay, tốc độ của các chương trình Java cũng không còn là điều quan trọng.

2.10 Đa tuyến (Multi-thread)

Java đã đặt ra từ đầu tính năng hỗ trợ lập trình đa tuyến.Tính năng này cho phép chúng ta có thể thực hiện nhiều tiến trình song song và tương hỗ với nhau tránh được tính tuần tự nhàm chán và những thời gian chết trong chờ đợi.

2.11 Linh động (flexible)

Vì dùng trình thông dịch nên Java là ngôn ngữ rất năng động.Trong khi chạy,môi trường Java có thể tự mở rộng để liên kết với các lớp mới có thể ở một máy chủ ở xa trên mạng (hay trên Internet).Đó chính là thuận lợi to lớn của Java so với C/C++.

[Chương 3]Cài đặt hệ JDK 1.4.1
Chương 3:Cài đặt hệ JDK 1.4.1

Ở mục này mình chỉ giới hạn cho việc cài đặt hệ Java JDK 1.3.1 trên nền Intel-Windows . Việc cài đặt trên các nền khác cũng tương tự nhưng đòi hỏi phải có bộ cài đặt tương ứng.

3.1 Chạy chương trình cài đặt

Tập tin jdk14-win32.exe trên đĩa CD bán sẵn là một tập tin cài đặt nén tự giải nén.Bạn cho nó tự chạy bằng dòng lệnh Run hoặc có thể kích đúp lên biểu tượng của nó tại Exprorer.Thư mục mà định là JDK cài đặt là C:\j2sdk1.4.0
Nếu bạn muốn cài đặt thêm các tài liệu của Java trong bộ jdk14-doc.exe ,hay bộ hướng dẫn tự học tutorial.exe cũng trên đĩa CD,bạn nên cài đặt chúng vào cùng thư mục chính C:\j2sdk1.4.0\docs

3.2 Cập nhật biến đường dẫn trong hệ điều hành
Để thuận tiện trong việc biên dịch và chạy các chương trình Java,mình đề nghị bạn cập nhật biến đường dẫn (PATH) để có thể chỉ đến thư mục chính của Java. Khi đó chúng ta có thể gọi các chương trình chạy được của Java từ bất kỳ thư mục nào.

Để xét biến PATH một cách thường trực,ta thêm đường đường dẫn đầy đủ (full patch đó) của thư mục Java C:\j2sdk1.4.0\bin vào biến PATH
này:

+Trong Window NT,bạn thực hiện như sau:
  • Khởi động Control Panel,chọn System,chọn Environment , click vào biến PATH trong User Variables and System Variables. Thêm vào bên phải của nội dung biến hiện dòng sau:
Mã:
;c:\j2sdk1.4.0\bin
  • Chú ý rằng cầu phải có dấu ; ở đầu
  • Bạn chỉ được có 1 thư mục Java Bin trong biến PATH hiện hành
  • Nhắp vào OK,SET hay APPLY để việc cập nhật có hiệu lực.Và kết quả có hiệu lực ngay khi bạn mở một cửa sổ lệnh mới.
+ Trong Windows 98/95 , bạn cần vào tập tin AUTOEXEC.BAT để thay đổi
  • Bạn chọn START,RUN và nhập dòng sysedit vào ô dòng lệnh, bấm OK (bạn cho chạy chương trình biên soạn của hệ thống SYSEDIT.EXE). Chọn cửa sổ chứa tập tin AUTOEXEC.BAT .
  • Tìm dòng khai báo biến PATH , nếu không có bạn thêm vào theo mẫu sau:
Mã:
    SET PATH=c:\j2sdk1.4.0\bin
  • Nếu có, bạn thêm vào bên phải dòng khai báo biến PATH,tương tự như ở WIndows NT,đoạn sau:
Mã:
;c"\j2sdk1.4.0\bin
  • Để việc cập nhật có hiệu lực,trong cửa sổ lệnh hiện hành,chạy dòng lệnh sau:
Mã:
c:\autoexec.bat
  • Kiểm tra lại tại chỗ bằng cách từ dấu nhắc dòng lệnh, gõ path, ta sẽ thấy giá trị của biến PATH

3.3 Kiểm tra biến CLASSPATH (nếu nâng cấp)

Biến CLASSPATH chỉ cho các ứng dụng Java biết thư mục chứa các Lớp của người dùng (ssẻ classes) mà Java sẽ tham chiếu đến.Ta có thể dùng tuỳ chọn-classpath trên dòng lệnh nhưng điều đó không thuận tiện lắm.Nếu bạn chưa bao giờ cài các bản Java JDK đời cũ,bạn có thể bỏ qua bước này. Bạn cũng có thể xoá sạch biến này nếu bạn đã từng cài ít nhất một lần JDK 1.1.x .

Thật sự ra khi làm việc với JDK 1.4 , bạn không phải lo lắng gì về biến CLASSPATH , kể cả khi biến này được gán cho bản JDK đời cũ. Tuy vậy, nếu trong đường dẫn của CLASSPATH chứa thư mục có tập tin classes.zip của bản đời cũ,thì bạn phải xoá các giá trị của CLASSPATH đi.

Để điều chỉnh giá trị của biến CLASSPATH , bạn thực hiện các bước tương tự như điều chỉnh biến PATH ở mục trên.Thêm vào đó là:
  • Xoá đi classes.zip ở thư mục được khai báo trong CLASSPATH . Hãy để nguyên các cài đặt của bất kỳ ứng dụng nào, và giữ cho thư mục làm việc là thư mục hiện hành.
  • Xoá hẳn biến môi trường CLASSPATH. Trong những trường hợp đặc biệt hãy khai báo tường tận trên dòng lệnh bằng đối sổ -classpath. Đối với JDK 1.4 , CLASSPATH có giá trị mặc định là thư mục hiện hành.
Khi bắt đầu viết và thực thi chương trình Java bạn sẽ hiểu rõ hơn về ý nghĩa của biến môi trường CLASSPATH.

3.4 Bắt đầu sử dụng các công cụ JDK

Đến đây máy của bạn đã sẵn sàng cho việc sử dụng JDK.Bạn có thể thực hiện một vài lệnh đơn giản để chắc chắn rằng chúng sẽ hoạt động tốt.

Bạn có thể thử với các trình biên dịch, thông dịch hay bất kỳ phần mềm công cụ nào bằng cách gõ vào dòng lệnh và không hỗ trợ giao diện đồ hoạ GUI, chỉ trừ AppletViewer (trình duyệt Applet) . Trong Windows NT/2000/XP bạn phải làm việc với các công cụ JDK trong cửa sổ lệnh Command,còn trong Windows 95/98 là cửa sổ DOS-Promt . NẾu muốn sử dụng môi trường trực quan,bạn có thể chọn cài đặt các hệ phát triển hỗ trợ Java như JBuilder , visual Age ... Trong bài học này phần lớn mình sử dụng trình soạn thảo Notedpad và biên dịch mã nguồn Java từ dòng lệnh , khi bạn đã hiểu thấy đáo về Java chúng ta sẽ chuyển qua sử dụng môi trường trực quan JBuilder.

Bạn hãy thử 2 chương trình chính của Java đó là:Trình biên dịch Java (javac.exe hay Java Compiler), trình thực thi ứng dụng Java (java.exe) đơn giản như sau:
Mở trình notepad của Windows.Nhập vào:
Mã:
public class HellOWorld
 {

                        System out println("HellOWorld");

Lưu file vào thư mục bất kỳ với tên HellOWorld.java, mở cửa sổ Command của Windows gọi trình biên dịch Java như sau:
Mã:
c:\j2sdk1.4.0>javacHellOWorld.java
Nếu cấu hình biến môi trường đúng,trình biên dịch javac sẽ dịch mã nguồn thành file.class .Bạn gọi trình diễn dịch và thực thi Java như sau:

Phần 3:Ngôn ngữ lập trình Java

Chương 1:Lập trình hướng đối tượng
1.1 Lập trình hướng đối tượng

Lập trình hướng đối tượng (OOP - Object Oriented Programming) là tư dauy của mô hình lập trình hiện đại . Hướng đối tượng tạo nên một cách suy nghĩ mới, thực tế hơn và linh hoạt hơn mô hình lập trình thủ tục truyền thống.Hướng đối tượng giúp lập trình viên dễ dàng mở rộng, bổ sung cho các chương trình của mình.
*Đối tượng
 Thật ra khái niệm đối tượng không xa lạ với lập trình viên. Hàng ngày , chúng ta đã làm việc với các đối tượng như:đồng hồ , máy tính , điện thoại , ... và tất cả các thực thể trong đời sống. Khi làm việc với một đối tượng , chúng ta không tách biệt số lượng và phương thức . Như khi ta mở radio vậy ta không cần chú ý đến số liệu (như tần số của đài phát sóng)), không chú ý đến phương thức (như làm sao có âm thanh , máy chuyển đài như thế nào ), mà ta chỉ cần mở công tắc điện , chọn đài và nằm xuống nghe nhạc.

Sử dụng OOP,ta cũng có sự đơn giản tương tự.Ta tạo một cấu trúc chứa dữ liệu và định nghĩa các phương thức để làm việc với dữ liệu.Như thế,đối tượng tạo và kết hợp dữ liệu bằng mã lệnh . Chúng ta có một đối tượng có đủ điều kiện để tồn tại và làm việc.

Trở lại với ví dụ trên,radio là một đối tượng với dũe liệu là hãng sản xuất,nguồn điện sử dụng, các băng tần,công suất loa ... và các phương thức là bật tắt công tắc nguồn , chuyển núm chọn băng tần , chuyển núm âm lượng , cắm tai nghe ... Đơn giản đối tượng Radio được định nghĩa trong ngôn ngữ lập trình với Java như sau:
Mã:
 public class Radio {
 /*định nghĩa dữ liệu đối tượng*/

/*Hành động của đối tượng*/

   speed=spead + volume ;
 }

 public void stop (){
 speed = 0 ;
}

      public void slowDown () {
                  speed = speed - volume
  } 
}

Trong trường hợp này, OOP xác định rõ ràng là thủ tục xuất dữ liệu ra màn hình là riêng của màn hình đó . Nghĩa là chúng ta có những đối tượng màn hình khác nhau.

Bởi vì sau khi bạn đã trừ tượng hoá đầy đủ các yếu tố của một đối tượng màn hình , lúc bạn viết thủ tục, các yếu tố cụ thể sẽ không còn ảnh hưởng đến thủ tục bạn viết , đối tượng màn hình sẽ tính toán giùm bạn.

  • Khi sử dụng đúng, OOP bắt buộc ta phải tổ chức mã lệnh thành nhiều đoạn nhỏ để dễ bề quản lý.
  • Vì dùng OOP , mỗi đoạn mã nhỏ đó sẽ được tổ chức rất tự nhiên không cần ta suy nghĩ nhiều về việc tổ chức.
Đây là một đặc điểm mạnh bất ngờ của OOP.

1.3 java và hướng đối tượng

Sự hỗ trợ lập trình hướng đối tượng là cốt lõi của Java . Mọi hoạt động , mọi câu lệnh của Java đều phải tác động lên và qua các đối tượng. Mà tượng trưng cho chúng là các Lớp (class) . Đối tượng là những mẫu (instance) của lớp . Trong ý nghĩa này,lớp có thể được xem như một cái khuôn đúc nên các đối tượng . Lấy hình chữ nhật làm ví dụ , ta có thể tạo ra một hình chữ nhật với toạ độ x,y của đỉnh thứ nhất, chiều dài , chiều rộng , cách di chuyển , phương thức đổi kích thước ... Ta tạo một lớp Rectangle , và viết mã lệnh cho nó. Như thế khi thực hiện các mã lệnh ta cần tạo ra một mẫu lớp này . Mẫu ấy chính là một đối tượng Rectangle cụ thể. Tương tự như trong ví dụ trên Radio là một lớp làm khuôn đúc ra các đối tượng như Sony,Panasonic.

  • Chia nhỏ mã lệnh thành đơn vị nhỏ nhất:

  • Xem xét các tính chất chung giữa các đơn vị nêu trên:

2.1 Chương trình Hello World - Xin chào Thế Giới
Như đã giới thiệu trong chương 1, chúng ta có rất nhiều loại chương trình Java , tuy nhiên chương trình Java dạng Console kết xuất văn bản ra màn hình là loại ứng dụng đơn giản nhất. Chương trình Console thường dùng để minh hoạ về ngôn ngữ Java rõ ràng , hiệu quả.Với phần bắt đầu tiếp cận Java này chúng ta hãy trở lại với chương trình hellOWorld.java trước đây. Chương trình HellOWorld là một chương trình kinh điểm dễ viết nhất và hầu nó nổi tiếng qua tất cả ngôn ngữ lập trình vì rất nhiều lý do:kiểm tra xem cấu hình hệ thống và môi trường lập trình có thiết lập đúng không , sử dụng đoạn mã chương trình đơn giản nhất. Kiểm tra cách biên dịch và thực thi chương trình ở mức tối thiểu (không cần thêm thư viện hay khai báo đặc biệt ) . Bạn mở trình notepad của Windows và nhập vào đoạn mã sau:
Mã:
  public class HellOWorld
 {
      public static void main (String[]args)
  {
          System.out.println ("hellOWorld") ;
          }
    }
Bạn lưu file này vào thư mục C:\Learning> với tên HellOWorld.java.Lưu ý quan trọng đó là tên file phải trùng với tên lớp chính xác theo cả chữ hoa và chữ thường. Ở đây nếu bạn lưu file với tên hellOWorld.java hay Helloworld.java thì quá trình biên dịch sẽ gặp rắc rối và trình biên dịch sẽ yêu cầu bạn đặt tên file trùng với tên lớp.Đây là điều đầu tiên bạn cần nhớ trước khi biên dịch Java. Lý do ở đây là trình biên dịch Java ban đầu được viết trên môi trường Unix của Sun mà Unix là môi trường không như Windows, UNix phân biệt kiểu chữ đối với tên file.

Bây giờ bạn mở cửa sổ Command của Windows và biên dịch file nguồn HellOWorld.java như sau:

Mã:
C:\Learing>javac HellOWorld.java
Với lệnh biên dịch trên bạn sẽ thu được file HellOWorld.class nằm cùng với thư mục File.class là file chứa mã biên dịch nhị phân để Java diễn dịch (còn gọi là bytecode)

Lưu ý:Ở đây javac nằm trong thư mục C:\j2sdk1.4.0\bin đã đuợc chúng ta thiết lập sẵn đường dẫn trong biến môi trường PATH nên bạn coóthể triệu gọi trình biên dịch javac bất cứ lúc nào.
Sau khi biên dịch xong bạn chạy file.class như thế nào? Muốn chạy các chương trình .exe của Microsoft bạn phải có DOS hoặc Windows. Tương tự muốn chạy chương trình .Class của Java bạn gọi trình Java từ dòng lệnh sau:
Mã:
C:\Learing>java HellOWorld
HellOWorld


Lưu ý: Ở đây bạn chỉ cần gõ tên lớp và không thêm đuôi .class phía sau tên lớp
