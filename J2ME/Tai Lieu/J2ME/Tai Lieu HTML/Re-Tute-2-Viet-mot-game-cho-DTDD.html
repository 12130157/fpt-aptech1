Chào các bạn,
Trong tute 2 này tui xin nói về:
1. xây dựng một game canvas
2. xây đựng một layer manager
3. xây dựng một GlassPane layer

Các bạn có thể download source code tute này trong share lib.
Note: Source code của mỗi tute hoàn toàn độc lập nên các bạn không cần phải download lại source của các tute trước.

Sơ đồ sau minh họa mối quan hệ giửa các lớp ta sẽ nói chi tiết trong tute 2 này


1. Game Canvas
Canvas là một đối tượng Displayable, có nghĩa là nó có thể được dùng để hiển thị một nội dung bất kì ra màn hành.
Trong MIDP 1, lớp Canvas có method paint(Graphics g) được khai báo abstract. Do đó khi tạo một canvas cho chương trình
ta tạo ra một lớp ví dụ như GameCanvas extend từ lớp Canvas này và đặt tất cả các thao tác vẽ vào method paint(Graphics g) của nó
import javax.microedition.lcdui.Canvas;
public class GameCanvas extends Canvas {
    ...
    protected void paint(Graphics g) {
        // Các thao tác vẽ ra màn hình đặt ở đây
    }
    ...
}
Thông thường ta không gọi method paint(Graphics g) trực tiếp. Lớp Canvas có cung cấp một method khác là repaint()
được dùng để yêu cầu canvas vẽ lại nội dung màn hình. Method repaint() này sẽ thay ta gọi method paint(Graphics g).
Nhưng có một điều các bạn lưu ý, sau khi gọi repaint nội dung của canvas không chắc được update ra màn hinh ngay lập tức.
Do đó nếu các bạn muốn chắc chắn nội dung canvas được update ra màn hình trước khi làm tiếp thao tác nào nữa các bạn dùng method
serviceRepaints() cũng do lớp Canvas cung cấp.
while (!m_destroyed) {
    canvas.repaint();
    canvas.serviceRepaints();
}
Khi implement method paint(Graphics g), nếu có nhiều thao tác vẽ, ta không nên vẽ thẳng vào canvas
vì khi đó đó sẽ gay ra hiện tượng màn hình bị giật rất khó coi. Để không bị hiện tượng này thường người ta dùng một màn hình ảo
gọi là offscreen để vẽ vào đó. Sau đó sẽ "đổ" toàn bộ nội dung của offscreen vào canvas.
protected void composeGraphics(Graphics g) {
    // Các thao tác vẽ vào offscreen
}

protected void paint(Graphics g) {
    composeGraphics(offscreen.getGraphics());
    g.drawImage(offscreen, 0, 0, Graphics.TOP | Graphics.LEFT);
}
Canvas còn có một số method nữa cho phép chương trình biết được các sự kiện phím bấm đó là:
protected void keyPressed(int keyCode);
protected void keyRepeated(int keyCode);
protected void keyReleased(int keyCode);
Chúng ta sẽ nói về cách sử lí input của user trong những tute sau.

2. Layer Manager
Một chương trình game phức tạp, thông thường người thiết kế sẽ chia công việc vẽ ra màn hình thành nhiều layer.
Mỗi layer đảm nhiệm một nhiệm vụ vẽ riêng. Khi chạy các layer này sẽ lần lượt được yêu cầu vẽ ra offscreen tạo thành một
thế giới game phức tạp. Và tất nhiên sau khi các layer hoàn thành nhiệm vụ,
offscreen sẽ được đổ trực tiếp ra màn hình như đã trình bày ở trên.

Tất cả các layer trong game đều phải implement interface ILayer như sau:
public interface ILayer {
    public void paint(Graphics g, int x, int y, int width, int height);
    public boolean isVisible();
}
Trong đó,
method paint được dùng gọi khi layer manager muốn vẽ lại một layer nào đó
nếu method isVisible() trả về true, layer này sẽ được vẽ, false, layer manager sẽ bỏ qua

Layer manager sẽ quản lí tất cả các layer trong một game.
Lớp này hỗ trợ các thao tác như add một layer mới, remove tất cả các layer hiện có.
Method paint được gọi để vẽ tất cả các visible layer. Các layer được vẽ theo thứ tự
FIFO tức là vào trước thì được vẽ trước
for (int i = 0; i < m_layerCount; i++) {
    if (m_layers[i]!=null && m_layers[i].isVisible()) {
        m_layers[i].paint(g, 0, 0, m_width, m_height);
    }
}
